Hosted Payment Page (HPP)

Hosted Payment Page (HPP) — це тип інтеграції, при якому клієнт перенаправляється на зовнішню платіжну сторінку платіжного шлюзу, де він самостійно вводить необхідні платіжні дані (наприклад, номер телефону, ПІБ, email, платіжний метод тощо).

Такий підхід дозволяє швидко запустити прийом або виплати коштів, без необхідності самостійно реалізовувати валідацію, безпеку та взаємодію з платіжними методами на стороні мерчанта.

Сфери використання​

Інтеграція HPP підходить як для:

  • PayIn — прийом платежів з карток, електронних гаманців, мобільних операторів тощо
  • PayOut — виплати клієнтам, які вводять необхідні реквізити на сторінці платіжного шлюзу

Переваги HPP інтеграції​

  • Швидка інтеграція — мінімальні технічні вимоги до бекенду мерчанта
  • Не потрібна реалізація платіжної логіки на стороні мерчанта
  • Безпечність — усі чутливі дані вводяться безпосередньо на захищеній платіжній сторінці
  • Універсальність — можна використовувати для широкого спектра платіжних методів
  • Можливість брендового кастомізації сторінки (опційно)

Недоліки HPP інтеграції​

  • Залучення користувача до процесу введення платіжних реквізитів
  • Менший контроль над UX/UI у порівнянні з повністю кастомним рішенням
  • Залежність від доступності платіжної сторінки шлюзу
  • Відсутність повної автоматизації — користувач має сам підтверджувати платіж або виплату

Коли варто обирати HPP?​

  • Потрібно швидко стартувати прийом або виплати коштів
  • Немає потреби в складній кастомній логіці або автоматизації
  • Хочете делегувати безпеку на сторону платіжного шлюзу
  • Потрібен уніфікований UX для різних платіжних методів
