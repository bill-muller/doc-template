Формат відповіді

Всі відповіді від API містять об'єкт error з даними про помилку (навіть якщо помилки не було), а також об'єкт response, якщо під час виконання запиту не виникло помилки. Об'єкта response може не бути у відповіді, якщо error.code не дорівнює 0.

Структура об'єкта response залежить від методу API і описана в документації кожного методу.

Нижче наведено опис структури об'єкта з помилкою:

Ім'яТипОпис
errorobjectДані про помилку
error.codeintegerКод помилки. Якщо під час обробки запиту помилок не виникло, код дорівнюватиме 0. Повний список кодів наведено у довіднику Коди помилок API
error.titlestringНазва помилки
error.messagestringДодатковий опис помилки (опціонально)

Приклад запиту​

Запити до API можуть містити як єдиний об'єкт auth, так і додаткові параметри відповідно до специфікації конкретного API методу.

{
    "auth": {
        "point": 111,
        "key": 1234,
        "hash": "fda3ff2903690ed6795ea997a8cc1773"
    },
    "id": 123456789
}
