Формат запиту

Всі запити до API платіжного шлюзу надсилаються у JSON форматі за допомогою методу POST.

У запитах обов'язково має бути заголовок Content-Type: application/json, інакше запит буде вважатися некоректним навіть при валідному JSON у ньому.

Авторизація​

Кожен запит, крім основних параметрів методу, повинен містити об'єкт auth з даними авторизації:

Ім'яТипОпис
authobjectДані авторизації
auth.pointintegerЦе ваш point_id з облікових даних, які отримані у менеджера
auth.keyintegerСесійний ключ, що створюється на стороні мерчанта Як значення ключа можна використовувати unixtime
auth.hashstringmd5 хеш рядка, отриманого шляхом об'єднання point_id, api_token та key: md5(point + api_token + key)
Приклад правильно порахованого hash (дані у прикладі вигадані):
point_id = 111
key = 1234
api_token = aiGha$et0ainah4%*eu9Vish8oo
Коректний hash: fda3ff2903690ed6795ea997a8cc1773

Приклад запиту​

{
    "auth": {
        "point": 111,
        "key": 1234,
        "hash": "fda3ff2903690ed6795ea997a8cc1773"
    }
}

Приклади коду для розрахунку auth.hash​

from hashlib import md5

def calculate_hash(point_id, api_token, key):
    return md5(f'{point_id}{api_token}{key}'.encode()).hexdigest()
