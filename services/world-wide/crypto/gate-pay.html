Gate Pay

Gate Pay – криптоплатіжне рішення, що дозволяє здійснювати швидкі та безпечні транзакції з низькими комісіями. Оплата відбувається шляхом сканування QR-коду на платіжній сторінці через мобільний гаманець Gate.io.

Порядок проведення платежу:

  1. Клієнт на сайті мерчанта обирає товар та даний метод оплати
  2. Мерчант створює транзакцію виконавши метод /transaction/create. У відповідь на запит мерчант отримає pay_url платіжної сторінки, на яку перенаправляє клієнта
  3. Клієнт на платіжній сторінці проходить реєстрацію та авторизацію в гаманці Gate.io. Якщо клієнт вже зареєстрований та авторизований, то йому одразу відобразиться QR-код для оплати
  4. На платіжній сторінці відображається QR-код, за допомогою якого клієнт може завершити оплату
  5. Мерчант очікує отримання колбека, а також може використати метод /transaction/find для отримання статусу платежу

Створення транзакції​

Для створення транзакції використовується метод API /transaction/create.

Всі запити до API платіжного шлюзу надсилаються у JSON форматі за допомогою методу POST. У запитах обов'язково має бути заголовок Content-Type: application/json, інакше запит буде вважатися некоректним навіть при валідному JSON у ньому.

Відображення платіжної сторінки​

Транзакцію можна вважати успішно створеною після отримання pay_url. Після цього клієнт може бути перенаправлений на вказаний URL.

Перенаправляти клієнта можна будь-яким зручним для вас способом:

  • Відобразити нашу сторінку в iframe на вашому сайті;
  • Виконати редирект клієнта з вашого сайту на отриманий pay_url;
  • Передати клієнту посилання на оплату іншими доступними методами.
Примітка

З метою протидії шахрайству для деяких мерчантів, нами може бути обмежений перелік доменів, на яких дозволено здійснення платежів. Якщо це обмеження активно, то при переході на сторінку оплати перевіряється заголовок Referer, що містить домен сайту, з якого виконано перехід.

В таких випадках платіжна сторінка може бути відображена клієнту тільки одним з наступних способів:

  • За допомогою iframe;
  • За допомогою JavaScript.

⚠️ При редиректі з кодом статусу HTTP 302 заголовок Referer не відправляється

Очікування завершення платежу​

Після того, як клієнт перейшов на платіжну сторінку, мерчант повинен дочекатись на колбек від платіжного шлюзу, який буде відправлено, як тільки транзакція отримає фінальний статус. Докладніше про колбеки можна прочитати в розділі Колбеки.

Важливо зазначити, що зворотній редирект клієнта на сторінку мерчанта не пов'язаний безпосередньо з відправленням колбека. Цілком можливо, що спочатку відбудеться редирект, а колбек надійде незабаром після цього. Це потрібно врахувати під час інтеграції.

Запит статусу платежу​

Мерчант може надіслати запит /transaction/find для отримання актуального статусу транзакції. Детальніше про метод /transaction/find можна прочитати на сторінці опису методу.

⚠️ Після успішного завершення транзакція отримає фінальний статус 1 (SUCCESS) (див. Довідник статусів).
